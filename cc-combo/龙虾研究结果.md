## 一、需求概述

### 1.1 核心目标
构建一个 CLI 工具，用于创建、管理和切换不同的 AI 编码工具配置（combo）。

### 1.2 第一阶段范围
仅支持 Claude Code，但架构需具备可扩展性。

### 1.3 可扩展性目标
后续可支持：
- OpenCode
- Codex
- GitHub Copilot CLI
- OpenClaw
- 其他 AI 编码工具

## 三、Claude Code 配置结构分析

### 3.1 主要配置文件

Claude Code 使用以下配置文件（来自 [JSON Schema](https://json.schemastore.org/claude-code-settings.json)）：

```
~/.claude/
├── settings.json          # 主配置
├── commands/              # 自定义命令
├── skills/                # 全局技能
├── .mcp.json              # MCP 服务器配置
└── plugins/               # 插件
```

### 3.2 settings.json 核心结构

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "env": {
    "DISABLE_TELEMETRY": "1",
    "DISABLE_ERROR_REPORTING": "1",
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "permissions": {
    "deny": ["Bash(rm -rf *)", "Read(~/.ssh/**)", ...]
  },
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [{ "type": "command", "command": "..." }]
      }
    ]
  },
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    }
  },
  "alwaysThinkingEnabled": true,
  "cleanupPeriodDays": 365,
  "enableAllProjectMcpServers": false
}
```

### 3.3 MCP 服务器配置 (`.mcp.json`)

位于项目根目录，格式：
```json
{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "@some/mcp-server"],
      "env": { "KEY": "value" }
    }
  }
}
```

### 3.4 Skills 技能配置

Skills 目录结构：
```
~/.claude/skills/
└── skill-name/
    └── SKILL.md
```

### 3.5 全局 CLAUDE.md

```
~/.claude/CLAUDE.md    # 全局项目说明
```

## 四、可扩展性研究

### 4.1 目标工具配置对比

| 工具 | 配置文件 | 配置格式 | 特点 |
|------|----------|----------|------|
| **Claude Code** | `~/.claude/settings.json`, `~/.claude/commands/`, `~/.claude/skills/` | JSON + Markdown | Agent teams, hooks, permissions |
| **GitHub Copilot CLI** | `~/.copilot/` | JSON | `@workspace` alias, 集成 GitHub |
| **OpenCode** | `~/.opencode/` 或项目 `.opencode/` | JSON | MCP 支持 |
| **Codex** | `~/.config/codex/` | JSON | OpenAI API |
| **OpenClaw** | `~/.openclaw/` | JSON + Markdown | Workspace, agents, skills |

### 4.2 统一配置抽象设计

建议使用**插件式架构**（Adapter Pattern）：

```
┌─────────────────────────────────────┐
│         cc-combo CLI (Core)         │
├─────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌───────┐ │
│  │ Claude  │  │ OpenCode│  │ Copilot│ │
│  │ Adapter │  │ Adapter │  │ Adapter│ │
│  └─────────┘  └─────────┘  └───────┘ │
├─────────────────────────────────────┤
│  Combo Manager (创建/切换/导出/导入) │
│  Profile Storage (本地文件)          │
└─────────────────────────────────────┘
```

### 4.3 Combo 概念设计

每个 Combo 包含：
```typescript
interface Combo {
  id: string;           // 唯一标识
  name: string;         // 显示名称
  description?: string; // 描述
  tool: ToolType;       // claude-code | opencode | copilot-cli | ...
  config: {
    settings?: object;      // 工具特定配置
    mcpServers?: object;    // MCP 服务器
    skills?: string[];       // 技能列表
    agents?: AgentConfig[];  // 全局代理/子代理配置
    plugins?: string[];      // 插件列表
    env?: Record<string, string>;  // 环境变量
    commands?: object;       // 自定义命令
  };
  createdAt: string;
  updatedAt: string;
}
```

### 4.4 Agents 配置详解

#### Claude Code Agents

Claude Code 支持 **Agent Teams**（多代理协作），配置文件位于：

```
~/.claude/agents/           # 用户级全局 agents
.claude/agents/             # 项目级 agents
```

Agent 配置结构示例：
```json
{
  "agents": {
    "reviewer": {
      "description": "代码审查代理",
      "model": "claude-opus-4-5",
      "instructions": "你是一个专业的代码审查员...",
      "tools": ["Read", "Bash", "Grep"],
      "permissions": {
        "deny": ["Bash(rm -rf *)"]
      }
    },
    "debugger": {
      "description": "调试代理",
      "model": "claude-sonnet-4",
      "mode": "agent"  // agent | ask | preview
    }
  }
}
```

settings.json 中启用 Agent Teams：
```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

#### OpenClaw Agents

OpenClaw 的 agents 存储在 `~/.agents/skills/` 和 `~/.openclaw/skills/`：

```
~/.agents/skills/           # 用户自定义 skills
~/.openclaw/skills/         # OpenClaw 内置 skills
~/.openclaw/workspace/      # 工作区配置
```

每个 skill 包含 `SKILL.md` 定义其行为。

#### Agents 与 Combo 的关系

```
Combo "security-audit"
├── settings.json        # 配置（可含 agent teams 启用标志）
├── agents/              # 全局 agents 配置
│   ├── reviewer.json
│   └── debugger.json
├── skills/              # 链接到全局 skills
├── mcps/                # MCP 服务器
└── commands/            # 自定义命令
```

每个 combo 可以包含自己的一套 agents 配置，切换 combo 时同步切换 agents。

## 五、技术选型建议

### 5.1 CLI 框架

| 框架 | Stars | 优点 | 缺点 |
|------|-------|------|------|
| **oclif** (Salesforce) | 高 | TypeScript 原生、插件系统、自动生成文档 | 学习曲线较陡 |
| **commander.js** | 高 | 简单易用、灵活 | 需手动处理参数解析 |
| **yargs** | 高 | 强大的参数解析、 自动生成 help | 配置较繁琐 |

**推荐**: **oclif** 或 **commander.js + 自建插件系统**

理由：
- oclif 有完善的插件架构（类似于我们需要的设计）
- TypeScript 原生支持
- Salesforce 维护，有保障

### 5.2 配置文件格式

**推荐**: `combo.json` + `settings.json` 分离

```
~/.config/cc-combo/
├── combos/
│   ├── default/
│   │   ├── combo.json        # Combo 元数据
│   │   ├── settings.json    # 工具配置
│   │   ├── agents/          # Agents 配置
│   │   ├── skills/          # 软链接到全局 skills
│   │   ├── mcps/            # MCP 服务器
│   │   └── commands/        # 自定义命令
│   ├── security-audit/
│   └── web-dev/
├── global/
│   ├── skills/              # 共享的全局 skills
│   └── mcps/                # 共享的 MCP 定义
├── active-combo -> combos/default  # 软链接
└── config.json              # cc-combo 自身配置
```

### 5.3 存储方案

- **本地 JSON 文件**: 简单直接，用户可控
- **可选: SQLite**: 便于查询和备份
- **不推荐**: 远程服务（增加复杂性和隐私顾虑）

## 七、关键设计决策

### 7.1 配置隔离策略

**方案 A**: 符号链接切换（推荐）
- 每个 combo 有独立 `~/.claude.d/<combo-name>/`
- 激活时将 `~/.claude` 符号链接到目标 combo

**方案 B**: 环境变量覆盖
- 使用 `CLAUDE_CODE_SETTINGS_PATH` 等环境变量
- 不修改本地文件，但需要每次启动时设置

### 7.2 API Key 安全

- **不存储明文**: 使用 `keytar` 或 `secret-service` 存储
- **可选: 加密存储**: 使用用户主密钥加密
- **环境变量注入**: 激活 combo 时注入到环境

### 7.3 MCP 服务器管理

MCP 服务器与 combo 解耦：
- MCP 服务器定义存储在 `mcps/` 目录
- combo 引用 MCP server 列表
- 同一 MCP server 可被多个 combo 共享

## 十、结论

### 10.1 可行性评估
**高可行性** - Claude Code 的配置结构清晰，已有类似项目可参考，TypeScript 生态成熟。

### 10.2 关键挑战
1. **配置隔离**: 需要优雅地处理多 combo 共存与切换
2. **安全存储**: API key 和敏感信息需要安全方案
3. **向后兼容**: 切换 combo 不应破坏现有单配置用户的体验

### 10.3 建议行动
1. **Phase 1**: 实现 Claude Code 的 combo 管理（MVP）
2. **Phase 2**: 添加模板系统和 MCP 管理
3. **Phase 3**: 定义 Adapter 接口，逐步添加 OpenCode/Copilot 支持
