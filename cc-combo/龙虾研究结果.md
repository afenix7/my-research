## 一、真实需求场景

### 1.1 核心痛点

老板的实际工作流涉及多个 Claude Code 生态框架：

| 框架 | 类型 | 典型用途 |
|------|------|----------|
| **gsd** | 项目管理框架 | Get Shit Done 任务执行 |
| **superpowers** | 开发者增强 | 技能扩展 |
| **gstack** | 工具栈 | 开发工具集成 |
| **ralph** | AI Agent | 自动化任务 |

**问题**：
- **同时启用多个 → 上下文污染**：gsd + superpowers + gstack + ralph 全部启用时，上下文窗口被稀释，每个框架的指令互相干扰
- **配置爆炸 → 上下文溢出**：agency-agents 有 100+ agents，全部加载会触发 Claude Code 的上下文长度警告
- **无法快速切换**：当前只能手动备份/恢复配置目录，效率低

### 1.2 类比：Docker 容器隔离

```
┌─────────────────────────────────────────────────────┐
│                    你的机器                          │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐           │
│  │ gsd     │  │superpow │  │ ralph   │  ← 同一   │
│  │ combo   │  │ers combo│  │ combo   │    时刻    │
│  └────┬────┘  └────┬────┘  └────┬────┘    只能   │
│       │            │            │          运行一个│
│       └────────────┴────────────┘
│              切换（停用一个，启另一个）
└─────────────────────────────────────────────────────┘
```

**核心需求**：像 `docker run <image>` 一样，一个命令切换整个工作环境。

```

├── settings.json                      # 主配置（JSON）

├── commands/                          # 自定义命令

│   ├── gsd/
│   ├── superpowers/
│   ├── gstack/
│   ├── ralph/
│   ├── agency-agents/                 # ← 100+ agents 所在
│   └── ...                            # 持续增长
├── agents/                            # Agent Teams 配置
│   └── reviewer.json
├── .mcp.json                          # MCP 服务器全局配置
└── plugins/                           # 插件
```

```json
  "alwaysThinkingEnabled": true,
  "cleanupPeriodDays": 365,
  "enableAllProjectMcpServers": false,
  "autoMemoryEnabled": true,
  "defaultMode": "execute"

```

### 3.3 Context Overflow 根源分析

```
当 combo 包含过多 components 时：
┌────────────────────────────────────────────────────┐
│ Claude Code 上下文窗口                              │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐│
│ │ settings │ │ skills/* │ │ agents/* │ │  mcps  ││
│ │  ~5KB    │ │ ~500KB+   │ │ ~300KB+  │ │ ~50KB  ││ ← 溢出
│ └──────────┘ └──────────┘ └──────────┘ └────────┘│
└────────────────────────────────────────────────────┘

Context 溢出警告示例：
"To avoid hitting context limits, consider reducing the number
of loaded skills or agents in your configuration."
```

**解决思路**：每个 combo 只包含**必要的最小子集**，而不是全局配置的全量叠加。

## 四、Combo 概念设计

### 4.1 Combo 定义

```typescript
interface Combo {
  id: string;               // 唯一标识 (kebab-case)
  name: string;             // 显示名称
  description?: string;    // 描述用途
  scope: 'global' | 'project';  // 作用域
  tool: 'claude-code';     // 第一阶段只支持 Claude Code
  config: {
    // 核心配置
    settings: ClaudeCodeSettings;  // settings.json 内容
    // 可选组件（均引用全局池中的 ID）
    skills?: string[];           // skill ID 列表
    agents?: string[];            // agent ID 列表
    hooks?: HookConfig[];         // hook 配置
    mcpServers?: string[];        // MCP server ID 列表
    commands?: CommandConfig[];   // 自定义命令
    // 环境
    env?: Record<string, string>;  // 环境变量（不含 secrets）
  };
  secrets?: string[];          // 引用的 secret key 列表（不存值）
  createdAt: string;
  updatedAt: string;
}
```

### 4.2 全局组件池

```typescript
// 全局技能池 ( ~/.config/cc-combo/global/skills/ )
interface GlobalSkillPool {
  'gsd': { name: string; path: string; description: string; };
  'superpowers': { name: string; path: string; description: string; };
  'ralph': { name: string; path: string; description: string; };
  // ... 全部注册在此
}

// 全局 Agents 池 ( ~/.config/cc-combo/global/agents/ )
interface GlobalAgentPool {
  'agency-agents': {
    name: string;
    path: string;
    description: string;
    agents: Record<string, { name: string; description: string; file: string; }>;
    // agency-agents 内部 100+ agents 的索引
  };
  'reviewer': { name: string; path: string; description: string; };
  // ...
}
```

### 4.2.1 Agency-Agents 粒度控制

核心需求：agency-agents 内的 agents 需要**亚群组（subgroup）控制**，支持三种模式：

```typescript
// Combo 内 agency-agents 的引用方式
interface AgencyAgentsRef {
  // 模式 A：全部使用
  | { mode: 'all' }

  // 模式 B：全部禁用
  | { mode: 'none' }

  // 模式 C：部分使用（精确选择）
  | { mode: 'select'; agents: string[]; }
}

// Combo config 中 agents 字段的扩展
interface ComboAgentsConfig {
  // 普通 agents（整包引用）
  single: string[];  // e.g. ["reviewer", "debugger"]

  // agency-agents 特殊处理
  agencyAgents: AgencyAgentsRef;
}
```

**TUI 展示示例**：

```
┌─ Select Agents ────────────────────────────────────────┐
│                                                          │
│  Single Agents:                                         │
│  [✓] reviewer (代码审查)                                 │
│  [ ] debugger (调试代理)                                │
│                                                          │
│  Agency-Agents Group (100+ agents):                     │
│  [A] All     [N] None     [S] Select subset             │
│                                                          │
│  Current: [S] Select subset                             │
│  Selected (3/100):                                       │
│    • reviewer-code (代码审查)                           │
│    • security-scan (安全扫描)                           │
│    • refactor-assist (重构辅助)                         │
│                                                          │
│  [Press A/N/S to change mode, SPACE to toggle agents]  │
└──────────────────────────────────────────────────────────┘
```

**实际存储**（combo.json 内）：

```json
{
  "agents": {
    "single": ["reviewer"],
    "agencyAgents": {
      "mode": "select",
      "agents": ["reviewer-code", "security-scan", "refactor-assist"]
    }
  }
}
```

**激活时展开**：切换 combo 时，cc-combo 将 `agencyAgents.mode=select` 的 agents 动态生成到 `~/.claude/agents/` 目录，只包含被选中的部分。


## 五、Hardened AI 操作环境

### 5.1 为什么需要 Hardened 环境

AI 生成 combo 时需要**操作本机上的 Claude Code/OpenCode/OpenClaw**：
- 读取现有配置
- 分析 skills/agents 效果
- 生成推荐配置

这涉及**在本地执行 AI 生成的脚本**，需要安全隔离。

### 5.2 Hardened 环境设计

```
┌─────────────────────────────────────────────────────────┐
│              Hardened AI Operation Env                   │
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │              AI Agent (cc-combo AI)                │ │
│  │  - system prompt: hardened security guidelines     │ │
│  │  - only allowed to read/write ~/.config/cc-combo/  │ │
│  │  - no direct shell access to production systems    │ │
│  └──────────────────────┬───────────────────────────────┘ │
│                         │                                │
│  ┌──────────────────────▼───────────────────────────────┐ │
│  │              Tool Access Layer                       │ │
│  │  - read_config(path)   ← 只允许 cc-combo 目录       │ │
│  │  - analyze_combo(id)   ← 分析给定 combo             │ │
│  │  - suggest_combo(desc) ← 基于描述生成建议           │ │
│  │  - validate_json()     ← 验证配置语法               │ │
│  │  - sandboxed_bash(cmd) ← 沙盒内执行（只读操作）     │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │              Target Tool Adapters                   │ │
│  │  - ClaudeCodeAdapter  (读 ~/.claude/)              │ │
│  │  - OpenCodeAdapter    (读 ~/.opencode/)            │ │
│  │  - OpenClawAdapter    (读 ~/.openclaw/)            │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 5.3 系统提示词核心原则

```
你是一个配置分析助手。你只能：
1. 读取 ~/.config/cc-combo/ 下的文件
2. 读取目标工具的配置文件（~/.claude/, ~/.opencode/, ~/.openclaw/）
3. 写入 ~/.config/cc-combo/ 下的文件
4. 在沙盒内执行只读命令（git status, ls, cat 等）

你禁止：
1. 执行任何修改操作（rm, mv, chmod 等）
2. 访问 ~/.ssh/, ~/.aws/, ~/.git-credentials 等敏感目录
3. 执行网络请求
4. 修改 ~/.claude/ 等工具的原始配置
```

### 5.4 AI Combo 生成流程

```
用户: "帮我分析我的 gsd 和 ralph 配置，生成一个适合代码审查的 combo"

1. AI 读取 ~/.claude/skills/gsd/SKILL.md
2. AI 读取 ~/.claude/skills/ralph/SKILL.md
3. AI 读取 ~/.claude/agents/ 下的相关 agents
4. AI 分析各配置的功能重叠和互补
5. AI 生成新 combo 建议：
   {
     name: "code-review",
     description: "适合代码审查的精简配置",
     skills: ["gsd/verify", "ralph/review"],
     agents: ["reviewer"],
     settings: { defaultMode: "plan" }
   }
6. 用户确认后写入 ~/.config/cc-combo/combos/code-review/
```

## 七、切换机制设计

### 7.1 切换命令

```bash
# 切换到指定 combo（原子操作）
cc-combo switch gsd-workflow

# 切换输出示例：
# ✓ Stopping current combo (empty)
# ✓ Activating gsd-workflow
# ✓ Linked ~/.claude → ~/.config/cc-combo/combos/gsd-workflow
# ✓ Linked ~/.claude/skills → ../../global/skills/gsd
# ✓ Linked ~/.claude/agents → ../../global/agents/reviewer
# ℹ️ Restart Claude Code to apply changes

# 查看当前
cc-combo current
# gsd-workflow

# 临时运行另一个 combo（不切换持久配置）
cc-combo run --combo security-audit -- claude
```

### 7.2 切换原子性保证

```typescript
async function switchTo(comboName: string) {
  const comboDir = path.join(COMBO_DIR, comboName);
  const claudeDir = process.env.HOME + '/.claude';
  const backupDir = claudeDir + '.backup.' + Date.now();

  // 1. 备份当前
  await fs.rename(claudeDir, backupDir);

  // 2. 创建新链接
  await fs.symlink(comboDir, claudeDir);

  // 3. 清理备份（异步，不阻塞）
  cleanupBackup(backupDir);
}
```

## 八、技术选型

### 8.1 CLI 核心框架

| 框架 | 推荐度 | 理由 |
|------|--------|------|
| **oclif** | ⭐⭐⭐⭐ | 插件架构完善，TypeScript 原生，Salesforce 维护 |
| **commander.js** | ⭐⭐⭐⭐ | 简单灵活，可自建插件系统 |

**推荐**: **oclif** + **@oclif/plugin-autocomplete**

### 8.2 TUI 框架

**推荐**: **blessed** + **blessed-contrib**

理由：TypeScript 原生、组件丰富（checkbox、list、form、table）、手感接近原生 terminal

### 8.3 存储

| 方案 | 适用场景 | 推荐度 |
|------|----------|--------|
| JSON 文件 | 简单直接、用户可控 | ⭐⭐⭐⭐⭐ |
| SQLite | 大量 combo、需要查询 | ⭐⭐⭐ |
| Yaml | 需要注释、人类可读 | ⭐⭐⭐ |

**推荐**: **JSON 文件**（`combo.json` 元数据 + `settings.json` 配置分离）

### 8.4 Secrets 管理

| 方案 | 平台 | 推荐度 |
|------|------|--------|
| **keytar** | 全平台 | ⭐⭐⭐⭐ |
| **secret-service** (Linux) | Linux | ⭐⭐⭐⭐ |
| **GnuPG** | 全平台 | ⭐⭐⭐ |

**推荐**: **keytar**（最终用户密钥加密存储）

## 十二、关键设计决策

### 12.1 为什么用软链接而非复制

| 方案 | 优点 | 缺点 |
|------|------|------|
| **软链接** | 节省空间、修改同步 | 需要相对路径处理 |
| **复制** | 完全隔离 | 占用空间、配置分裂 |

**选择**: 软链接 + 池化复用（多个 combo 共用同一个 skill/agent 源）

### 12.2 为什么 AI 操作需要 hardened 环境

直接让 AI 操作本机配置存在风险：
- AI 生成的代码可能破坏现有配置
- prompt injection 攻击可能窃取 secrets
- 误操作可能删除重要文件

**解决**: 强制 AI 只在 `~/.config/cc-combo/` 范围内操作，且只能执行白名单命令

### 12.3 项目级 combo 继承规则

```
项目有 .cc-combo/ → 使用项目的 combo
项目无 .cc-combo/ → 使用全局 active combo
combo 内无某组件 → 继承全局默认值或空
```
