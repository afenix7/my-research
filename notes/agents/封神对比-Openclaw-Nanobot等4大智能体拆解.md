# 封神对比：Openclaw、Nanobot 等 4 大智能体拆解

> 来源：视频字幕《封神对比！Openclaw_Nanobot 等 4 大智能体拆解，吃透通用智能体技术内核》

---

## 概述

本课程对比拆解市面上 4 款主流**通用智能体**框架，分析其架构设计与工程实现，目标是让开发者理解通用智能体的技术内核，从而能够基于这些框架构建自己的**垂直业务智能体**。

**4 大智能体：**
1. **Openclaw**（Open Cloud / Open Code，国外开源通用智能体）
2. **Nanobot**（NT Bot，通用智能体框架）
3. **Zeroclaw / NT Cloud**（第三个框架，同类衍生产品）
4. **Lobster AI**（网易大龙虾，国内代表产品）

核心论点：后三个框架均以 Openclaw 的设计思想为基础衍生，四者存在大量共性。

## 核心内容

### 1. Computer Use 不是新技术

- OpenAI 在更早期就已推出 **Computer Use** 相关能力
- **Anthropic** 在 2025 年 2 月收购了一家专门做**客户端智能体**的公司
- **Manus** 之前已有大量公司在尝试做 Computer Use
- Openclaw 等产品的爆火（GitHub Star 超越 Linux、React 等顶级项目），反映的是七八十% 的人对该技术向往但理解不深

**结论：** 这不是新技术，但它是一个**不错的工程化产品**——旧的技术思路，新的工程落地方案和应用思想。

### 2. 两种不同的智能体实现路径

| 路径 | 代表 | 核心技术 |
|------|------|----------|
| **LLM 路径** | Openclaw、Nanobot 等 | 语言大模型（LLM）+ 多模态模型 |
| **计算机视觉路径** | Anthropic 收购的公司 | 计算机视觉 + 大模型结合，从底层突破电脑控制权 |

两种路径殊途同归，最终目标一致：**让智能体接管电脑，帮助用户完成大量任务**。

### 3. 通用智能体 vs 垂直领域智能体

**通用智能体的定位误区：** 很多用户错误期望通用智能体能做所有事（炒股、设计 PPT、售卖产品等），实际上通用智能体只做**通用的事情**，本身没有垂直业务方法论。

**平台生态类比：**
- 通用智能体平台 ≈ 钉钉 / 飞书平台
- 垂直智能体服务商 ≈ 钉钉 / 飞书上的 ISV（独立软件服务商）
- 开发者应该**基于通用平台构建自己的垂直业务智能体**，而非期望通用智能体包揽一切

**垂直业务示例：**
- 养老行业专属智能体（老龄化社会需求）
- 金牌销售方法论智能体（将销售 SOP 封装为 Skills）

### 4. Plugin 与 Skills 扩展机制

通用智能体的核心拓展点有两个：

#### Plugin（插件）
- 用于扩展**外部服务连接能力**
- 典型案例：将 Openclaw 接入国内飞书（社区开发者贡献）
- 典型案例：将 Openclaw 接入企业内部数据库，实现数据库管理与操作
- 开发方式：开发一个 plugin 装入智能体，赋予其与外部系统交互的能力

#### Skills（技能）
- 用于封装**垂直业务方法论**
- 通用智能体不具备领域方法论，Skills 是补充垂直专业能力的手段
- 典型案例：将"金牌销售方法论"封装为 Skills，让智能体在销售场景中使用该方法论
- 社区中有专门设计用于帮助用户找到更好 Skills 的 Scout 工具

### 5. 手机智能体与权限问题

- 操作系统级别的手机智能体**技术上已实现**
- 核心障碍：**权限问题**——抖音、小红书等平台不一定允许智能体操作（可能破坏内部机制）
- 2025 年 2 月已出现一批手机智能体产品

### 6. 课程学习目标

1. 了解 4 个框架的**核心架构与设计理念**
2. 掌握通用智能体的**关键技术要素与核心组件**
3. 对比**不同架构的优劣势与适用场景**
4. 总结**通用智能体的核心开发模式**

**核心价值主张：** 在 AI Coding 时代（Codex、Claude Code 等工具加持下），只要理解架构设计思想，即使没有完整技术背景的人也能开发出自己的智能体产品。

## 4 大智能体架构对比表

> 注：以下对比基于课程描述及行业已知信息整理

| 维度 | Openclaw | Nanobot | Zeroclaw / NT Cloud | Lobster AI（网易大龙虾） |
|------|----------|---------|----------------------|--------------------------|
| **起源** | 国外开源，原创架构 | 基于 Openclaw 思想衍生 | 基于 Openclaw 思想衍生 | 国内产品，基于同类思想 |
| **定位** | 通用智能体框架 | 通用智能体框架 | 通用智能体框架 | 通用智能体，国内生态 |
| **主要技术** | LLM + 多模态 | LLM | LLM | LLM |

## 关键术语表

| 术语 | 含义 |
|------|------|
| **通用智能体（Universal Agent）** | 能够接管电脑执行多种通用任务的 AI 智能体，不专属于某一垂直领域 |
| **垂直领域智能体（Vertical Agent）** | 专注于特定行业或场景的智能体，内嵌领域方法论 |
| **Computer Use** | AI 通过操作系统接口控制计算机完成任务的能力 |
| **Plugin（插件）** | 为智能体添加外部服务连接能力的扩展模块（如飞书、数据库） |
| **Skills（技能）** | 将垂直业务方法论封装为智能体可调用的能力模块 |
| **LLM（大语言模型）** | Large Language Model，通用智能体的核心 AI 引擎 |
| **多模态模型** | 能处理文本、图像等多种输入的 AI 模型，增强 Computer Use 能力 |
| **计算机视觉路径** | 不依赖 LLM 理解，而用图像识别技术直接解析屏幕内容来操作电脑 |
| **Scout** | 社区中用于帮助发现和找到更好 Skills 的工具 |
| **Codex / Claude Code** | AI Coding 工具，降低智能体开发门槛 |
| **ISV（服务商）** | Independent Software Vendor，在平台生态上构建垂直服务的第三方开发者 |
| **Manus（Minus）** | 早期出现的 AI 计算机操作产品，先于 Openclaw 出现 |

## 总结

本课程的核心思路可以归纳为三层：

**第一层：认知校正**
- Openclaw 不是革命性新技术，Computer Use 早已存在
- 通用智能体不是万能的，不要对它有过高期望

**第二层：架构理解**
- 4 大框架共享相同的核心设计理念，Openclaw 是原型
- Plugin 负责连接外部世界，Skills 负责封装领域知识
- 两者结合，才能将通用智能体打造成垂直专业工具

**第三层：机会判断**
- 通用智能体平台由大厂主导（阿里通义、网易等）
- 开发者的机会在于：**基于通用平台，构建垂直业务智能体**
- 类比钉钉/飞书生态——平台提供基础能力，开发者提供垂直价值
- AI Coding 工具（Codex、Claude Code）极大降低了实现门槛

> 核心行动指南：理解架构设计 → 明确垂直场景 → 用 Plugin 接入数据 → 用 Skills 注入方法论 → 构建你的专属垂直智能体
