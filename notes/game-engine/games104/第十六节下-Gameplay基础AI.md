## Steering 系统

**Steering** 是寻路管道的第三阶段，解决的问题是：**AI 体如何按照物理约束真实地沿路径运动**。

即使 A* 找到了"最优路径"，一辆车或一个人也不可能精确沿折线瞬移——它有加速度、减速度、转弯半径等约束。Steering 系统将这些物理约束纳入运动计算，使 AI 体的移动看起来自然真实。

### 三类基本 Steering 行为

Reynolds（Craig Reynolds） 是 Steering 系统和群体行为系统（Boids/Swarm）的提出者。

#### 1. Seek / Flee（追逐 / 逃离）

$$\vec{a}_{\text{seek}} = \text{normalize}(\vec{p}_{\text{target}} - \vec{p}_{\text{self}}) \times a_{\max}$$

**变种：**
- **Pursuit（预测追踪）**：预测目标的未来位置，追踪预测点而非当前位置
- **速度场（Velocity Field）**：空间中定义方向场，每个质点跟随场方向运动

#### 2. Velocity Matching / Arrival（速度匹配 / 到达）

核心问题：AI 体从起点出发后需要**加速 → 到达目标前减速 → 恰好停在目标点**。

$$\frac{1}{2} a t^2 = d \implies t = \sqrt{\frac{2d}{a}}$$

工程实践：**每帧假设目标做匀速直线运动**，算出当前帧的加速度；下一帧根据目标新位置重新计算。

#### 3. Alignment（朝向对齐）

AI 体的朝向需要平滑地转向目标方向，而非瞬间旋转。

- 类比线速度的加减速，角速度同样需要**角加速度**和**角减速度**
- 遗漏角加速/减速会使 AI 体出现机器人式的"当当当"瞬间转向

### Steering 约束参数

| 参数 | 说明 |
|------|------|
| **最大线速度（max velocity）** | AI 体能达到的最大速度 |
| **最大线加速度（max acceleration）** | 最大加速 / 减速能力 |
| **最大角速度（max angular velocity）** | 最大转向速度 |
| **最大角加速度（max angular acceleration）** | 最大角加速 / 减速能力 |

## 群体模拟

**Crowd Simulation** 的应用场景：城市行人、鸽子群、RTS 大批小兵、数字孪生人群疏散仿真。

### 三类群体模拟方法

#### 1. 微观方法（Microscopic / Bottom-up）

代表算法：**Reynolds Boids** — 三条简单规则，群体行为自然涌现：

| 规则 | 说明 |
|------|------|
| **分离（Separation）** | 离邻居太近时产生斥力，防止重叠 |
| **内聚（Cohesion）** | 离群体过远时产生引力，保持群聚 |
| **对齐（Alignment）** | 与邻居保持相同的运动方向 |

**优点：** 规则极简，涌现行为自然（鱼群、鸟群效果逼真）
**缺点：** 结果不可控，不适合高度规则化场景

#### 2. 宏观方法（Macroscopic / Top-down）

典型实现（UE5 Mass AI）：
- 定义**车道（Lane）**和**区域图（Zone Graph）**
- 每个 AI 个体分配到一条 Lane，沿 Lane 方向移动
- 十字路口处定义 Lane 间的分叉和转换规则

**优点：** 行为高度可控，支持城市街道等规则场景

#### 3. 混合方法（Hybrid）

将宏观控制与微观个体行为结合：
- **宏观层**：定义群体的整体目标方向
- **微观层**：每个个体使用 Rule-based 方法自主进行局部避障和行为决策

典型应用：RTS 游戏（《星际争霸》、《帝国时代》）

### 力场方法（Force-based）

对障碍物进行**距离场（Distance Field）**计算：
- 每个障碍物周围建立距离场：距离越近，产生越强的斥力
- AI 个体在前进方向上叠加来自障碍物的斥力向量
- 计算成本远低于对每个个体运行完整寻路
- 适合数千个体的大规模群体模拟

## 碰撞回避

### Velocity Obstacles（VO，速度障碍法）

基本思想：预测到"沿当前速度前进会与某个移动物体碰撞"，则在**速度空间（Velocity Space）**中该物体形成一个**锥形障碍（VO Cone）**，调整速度避开该 VO 锥。

**局限：** VO 中只有我一方主动让路，对方不考虑让路。

### Reciprocal Velocity Obstacles（RVO，互惠速度障碍）

改进：**双方都承担一半责任**

- A 看到即将与 B 碰撞 → A 往左调整半份速度
- B 同时察觉将与 A 碰撞 → B 也往左调整半份速度
- 双方互相让路，结果是自然地靠同侧通行

**问题：** 大量个体时两两之间的 RVO 会产生冲突。

### Optimal RCA（ORCA，最优互惠碰撞回避）

ORCA 是 RVO 的全局优化版本：
- 将所有个体的速度约束在速度空间中表示为锥形区域
- 对所有速度锥求**明可夫斯基和（Minkowski Sum）**
- 在可行速度集合中求解对所有个体**最公平**的速度选择

**工程现实：** ORCA 实现复杂，计算成本高。**力场（Force-based）方法**因其简单性在工程中更受推荐——极简实现，参数调整直观，对大多数游戏场景完全够用。

## 碰撞回避系统在引擎中的定位

> "碰撞回避是一个独立的专用系统，而非 AI 决策系统的一部分。"

架构分层：

```
AI 决策系统（FSM / 行为树）
    ↓  发出移动意图
Steering 系统
    ↓  生成期望速度
碰撞回避系统（Force-based / RVO / ORCA）
    ↓  修正最终速度，避免 overlap
物理 / 运动执行
```

**Decoupling（解耦）** 是游戏引擎系统设计的核心原则：
- Event 系统：Publisher 发出消息时，不应假设谁来处理、处理顺序如何
- 依赖消息处理顺序的逻辑 = 隐含的强耦合
- 为 Event 系统设置优先级 = 隐含了处理顺序依赖 = 耦合增强

## 总结

1. **Steering 系统**：三类核心行为（Seek/Flee、Velocity Matching/Arrival、Alignment）都涉及加速度约束——线加速度 + 角加速度是实现自然运动的关键。
2. **群体模拟**：微观（Boids）涌现自然群体行为；宏观（Lane/Zone）适合城市行人；混合方案适合 RTS；力场方法低成本处理大规模群体。
3. **碰撞回避**：VO → RVO → ORCA 层层改进；工程实践中力场方法因简单性更受推荐；碰撞回避是独立于 AI 决策系统的专用模块，分层解耦是架构设计的核心原则。
