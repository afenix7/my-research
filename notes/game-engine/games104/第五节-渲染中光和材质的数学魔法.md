## 概述

渲染的三大主角：
1. **光（Light）**：光子的发射、反射、吸收，最终进入眼睛的全过程

## 光的物理模型

### 辐射度量学基础

#### 辐亮度（Radiance）

**Radiance** $L$ 是描述光场的最基本物理量，定义为：

$$L(x, \omega) = \frac{d^2\Phi}{dA\, \cos\theta\, d\omega}$$

- $\Phi$：辐射通量（Radiant Flux / Power），单位 W
- $dA$：面积元素
- $d\omega$：立体角元素
- $\cos\theta$：入射光方向与表面法线夹角的余弦

直觉理解：Radiance 是"沿某一方向单位立体角、单位投影面积的功率"。相机或眼睛测量的正是 Radiance。

#### 辐照度（Irradiance）

**Irradiance** $E$ 是表面单位面积接收到的总辐射通量：

$$E(x) = \int_{\Omega^+} L_i(x, \omega_i)\, (\mathbf{n} \cdot \omega_i)\, d\omega_i$$

**为什么要乘 $\cos\theta$（即 $\mathbf{n} \cdot \omega_i$）？**

这是非常朴素的几何事实：光斜射时，同样的光束照射更大面积，单位面积接收能量降低。类比现实：地球两极寒冷，就是因为太阳光以极小的 $\cos\theta$ 照射，单位面积接收能量少。

### BRDF（双向反射分布函数）

**BRDF**（Bidirectional Reflectance Distribution Function）描述：给定入射方向 $\omega_i$，出射方向 $\omega_o$ 的反射能量比例：

$$f_r(x, \omega_i, \omega_o) = \frac{dL_o(x, \omega_o)}{dE_i(x, \omega_i)} = \frac{dL_o(x, \omega_o)}{L_i(x, \omega_i)\, (\mathbf{n} \cdot \omega_i)\, d\omega_i}$$

BRDF 需满足两个物理约束：

1. **互易性（Helmholtz reciprocity）**：$f_r(x, \omega_i, \omega_o) = f_r(x, \omega_o, \omega_i)$

2. **能量保守（Energy conservation）**：

$$\forall\, \omega_i, \quad \int_{\Omega^+} f_r(x, \omega_i, \omega_o)\, (\mathbf{n} \cdot \omega_o)\, d\omega_o \leq 1$$

> 注意：这里是"能量保守（conservative）"而非"能量守恒（conserved）"——出射能量必须**小于等于**入射能量（光会被表面吸收），不能出射多于入射。如果违反此约束，在光线追踪中会产生能量放大，导致白色爆炸等错误。

## 渲染方程

### Kajiya 渲染方程（1986）

图形学宗师 **James Kajiya** 于 1986 年提出了渲染方程（The Rendering Equation），一个方程描述全部渲染：

$$\boxed{L_o(x, \omega_o) = L_e(x, \omega_o) + \int_{\Omega^+} f_r(x, \omega_i, \omega_o)\, L_i(x, \omega_i)\, (\mathbf{n} \cdot \omega_i)\, d\omega_i}$$

**各项解释：**

| 符号 | 含义 |
|------|------|
| $L_o(x, \omega_o)$ | 在点 $x$ 沿方向 $\omega_o$ 的出射辐亮度（相机看到的） |
| $L_e(x, \omega_o)$ | 点 $x$ 自发射的辐亮度（发光体） |
| $f_r(x, \omega_i, \omega_o)$ | BRDF，描述材质如何散射光 |
| $L_i(x, \omega_i)$ | 从方向 $\omega_i$ 入射的辐亮度 |
| $(\mathbf{n} \cdot \omega_i)$ | Lambert 余弦项 |
| $\int_{\Omega^+} \cdots d\omega_i$ | 对上半球所有入射方向积分 |

### 渲染的三大挑战

这个方程在数学上完美，但在实时渲染中极难求解，挑战主要来自三方面：

1. **光照复杂性**：光来自四面八方（球面积分），包含直接光、间接光（Color Bleeding）、折射、多次弹射等。

2. **材质复杂性**：BRDF 可以是任意复杂的函数，难以解析求解。

3. **可见性（Visibility）**：积分中的 $L_i$ 还需要考虑遮挡——即 Shadow。

## 经典光照模型（早期方案）

在物理精确渲染普及之前，早期游戏使用"三件套"方案，用最简单的方法近似渲染方程：

### 三件套

1. **Ambient Light（环境光）**：用一个常数代表所有间接光照
   $$L_{ambient} = k_a \cdot I_a$$

2. **Dominant Light（主光源）**：单一方向光或点光源
   - 方向光（Directional Light）：平行光，如太阳
   - 点光源（Point Light）：向四周发光，衰减 $\propto 1/r^2$
   - 聚光灯（Spotlight）：圆锥形光

3. **Environment Map（环境贴图）**：用 Cube Map 存储来自四面八方的高频光照，用于镜面反射

### Blinn-Phong 材质模型

**Blinn-Phong** 是最经典的经验光照模型，简单高效：

$$L = L_{diffuse} + L_{specular} + L_{ambient}$$

**漫反射项**（Lambertian diffuse）：
$$L_{diffuse} = k_d \cdot I \cdot \max(0, \mathbf{n} \cdot \mathbf{l})$$

**高光项**（Blinn-Phong specular）：
$$L_{specular} = k_s \cdot I \cdot \max(0, \mathbf{n} \cdot \mathbf{h})^p$$

其中 $\mathbf{h} = \text{normalize}(\mathbf{l} + \mathbf{v})$ 为半程向量，$p$ 为光泽度（shininess）。

**Blinn-Phong 的缺陷：**
- **能量不守恒**：通过调整 $k_d$、$k_s$、$p$ 可以使出射能量超过入射能量，在全局光照（如光线追踪）中会导致光能无限放大
- **不真实感**：所有材质都有"塑料感"，无法区分金属/非金属、粗糙/光滑的细节

### Shadow Map（阴影贴图）

**Shadow Map** 是解决可见性问题的经典算法：

1. 从光源视角渲染场景，将深度存入 Shadow Map 纹理
2. 主渲染时，将世界坐标变换到光源空间，比较深度值
3. 若当前点深度 > Shadow Map 中存储深度 → 在阴影中

**已知问题**：
- **Shadow Acne（自阴影）**：数值精度问题，需加 bias 偏移
- **Peter Panning**：bias 过大导致阴影与物体脱离
- **锯齿**：分辨率不足时边缘锯齿
- **长影子问题**：光源角度接近地平线时（清晨/傍晚），Shadow Map 覆盖范围不足，需使用多套算法过渡

## PBR 材质模型

### 从经验到物理

过去十几年，3A 游戏行业驱动下，材质系统经历了从 Blinn-Phong 到 **PBR（Physically Based Rendering）** 的革命性转变。

### 微平面理论（Microfacet Theory）

PBR 的核心假设：**任何表面都由无数微小镜面（micro-facet）组成**，每个微平面自身是完美镜面，只是法向方向不同。

- **粗糙（Rough）表面**：微平面法向散乱，光反射向各个方向，高光宽而柔
- **光滑（Smooth）表面**：微平面法向集中，光反射集中，高光窄而锐
- **完美镜面**：所有微平面法向相同，成为完美反射

这个简单假设统一解释了从漫反射到镜面反射的全谱材质表现。

### Metallic/Roughness 工作流

现代引擎最常用的 **金属度/粗糙度** 工作流参数：

| 参数 | 含义 | 范围 |
|------|------|------|
| **Base Color / Albedo** | 基础颜色（非金属时为漫反射色，金属时为镜面颜色） | [0, 1] |
| **Metallic** | 金属度（0 = 非金属，1 = 金属） | [0, 1] |
| **Roughness** | 粗糙度（0 = 完美镜面，1 = 完全粗糙） | [0, 1] |
| **Normal Map** | 切线空间法线贴图 | — |
| **AO** | 环境光遮蔽（Ambient Occlusion） | [0, 1] |

**金属 vs 非金属的物理区别：**

- **金属（Metallic）**：自由电子能捕获光子，光线几乎全反射，无漫反射（或极弱），反射颜色 = Base Color
- **非金属（Dielectric）**：光线进入材质内部，多次折射后以漫散射方式离开，产生漫反射颜色

因此 $F_0$（法线方向的菲涅尔反射率）：
- 非金属：$F_0 \approx 0.04$（约 4%，大多数介质的近似值）
- 金属：$F_0 = \text{Base Color}$（金属特有的彩色反射）

实现中：
$$F_0 = \text{mix}(\text{vec3}(0.04), \text{baseColor}, \text{metallic})$$

## BRDF 模型

### Lambert 漫反射 BRDF

最简单的漫反射 BRDF，表示光从表面各向同性地散射：

$$f_{Lambert}(\omega_i, \omega_o) = \frac{c}{\pi}$$

其中 $c$ 为 albedo（漫反射率）。除以 $\pi$ 来归一化，使能量守恒（对半球积分结果为 $c$，满足能量保守）。

> **常见 Bug 警告**：初学者常忘记除以 $\pi$，导致场景整体偏亮约 3 倍。

### Cook-Torrance 微平面 BRDF

镜面反射部分使用 **Cook-Torrance** 模型：

$$f_{spec} = \frac{D(\mathbf{h})\, F(\mathbf{v}, \mathbf{h})\, G(\mathbf{l}, \mathbf{v})}{4\, (\mathbf{n} \cdot \mathbf{l})\, (\mathbf{n} \cdot \mathbf{v})}$$

分母 $4 (\mathbf{n} \cdot \mathbf{l})(\mathbf{n} \cdot \mathbf{v})$ 是归一化因子，来自立体角变换的 Jacobian。

三个子项 **D、F、G** 各对应一种光学现象：

#### D — 法线分布函数（Normal Distribution Function, NDF）

描述微平面法向的统计分布，决定高光的形状与宽度。

**GGX（Trowbridge-Reitz）模型**（工业界标准）：

$$D_{GGX}(\mathbf{n}, \mathbf{h}, \alpha) = \frac{\alpha^2}{\pi \left[(\mathbf{n} \cdot \mathbf{h})^2 (\alpha^2 - 1) + 1\right]^2}$$

其中 $\alpha = \text{roughness}^2$（通常对粗糙度做平方映射，更符合感知线性）。

**GGX 的特点（被业界广泛采用的原因）：**
- **高频部分（高光核心）**：比 Phong/Beckmann 更尖锐、更亮
- **低频部分（高光边缘）**：有更长的"拖尾（long tail）"，不像其他模型快速衰减为零

类比音响：GGX 既有"清脆的高音"又有"沉厚的低音"，表现力最强。

```glsl
float D_GGX(vec3 N, vec3 H, float roughness) {
    float a      = roughness * roughness;
    float a2     = a * a;
    float NdotH  = max(dot(N, H), 0.0);
    float NdotH2 = NdotH * NdotH;

    float num   = a2;
    float denom = (NdotH2 * (a2 - 1.0) + 1.0);
    denom = PI * denom * denom;

    return num / denom;
}
```

#### F — 菲涅尔项（Fresnel Term）

**菲涅尔现象**：当视线掠过表面（夹角趋近 90°）时，反射率急剧增加。生活例子：远处马路上车的倒影、平视水面时看到倒影。

**Schlick 近似**（计算高效的近似公式）：

$$F(\mathbf{v}, \mathbf{h}) = F_0 + (1 - F_0)(1 - \cos\theta)^5$$

其中 $\cos\theta = \mathbf{v} \cdot \mathbf{h}$（观察方向与半程向量的点积）。

"**为什么是 5 次方？**"——讲师坦承未亲自推导，是前人推算出的近似，业界公认足够精确。

```glsl
vec3 F_Schlick(float cosTheta, vec3 F0) {
    return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
}
```

#### G — 几何遮蔽项（Geometry / Masking-Shadowing Term）

描述微平面之间相互遮挡的概率——粗糙表面微平面凹凸，会有部分微平面被其他微平面遮挡（masking）或投射阴影（shadowing）。

**Smith 近似**（分别处理 light 和 view 方向）：

$$G(\mathbf{l}, \mathbf{v}) = G_1(\mathbf{l}) \cdot G_1(\mathbf{v})$$

**Schlick-GGX 的 $G_1$ 子项**：

$$G_1(\mathbf{v}) = \frac{\mathbf{n} \cdot \mathbf{v}}{(\mathbf{n} \cdot \mathbf{v})(1 - k) + k}, \quad k = \frac{(\alpha+1)^2}{8}$$

关键设计亮点：**G 和 D 使用同一个 $\alpha = \text{roughness}^2$ 参数**，所以只需设置一个 roughness 参数，D 和 G 自动保持物理一致。

```glsl
float G_SchlickGGX(float NdotV, float roughness) {
    float r = (roughness + 1.0);
    float k = (r * r) / 8.0;
    return NdotV / (NdotV * (1.0 - k) + k);
}

float G_Smith(vec3 N, vec3 V, vec3 L, float roughness) {
    float NdotV = max(dot(N, V), 0.0);
    float NdotL = max(dot(N, L), 0.0);
    return G_SchlickGGX(NdotV, roughness) * G_SchlickGGX(NdotL, roughness);
}
```

### 完整 PBR Shader 伪代码

```glsl
// PBR Shading Pseudocode (GLSL-like)
// Direct lighting + IBL indirect lighting

vec3 baseColor = texture(albedoTex, uv).rgb;
float metallic  = texture(metallicTex, uv).r;
float roughness = texture(roughnessTex, uv).r;
vec3  N         = normalFromMap(normalTex, uv);
vec3  V         = normalize(cameraPos - worldPos);
vec3  R         = reflect(-V, N);

float alpha = roughness * roughness;
vec3  F0    = mix(vec3(0.04), baseColor, metallic);

// -------- Direct Light --------
vec3 Lo = vec3(0.0);
for (int i = 0; i < numLights; ++i) {
    vec3 L       = normalize(lightPos[i] - worldPos);
    vec3 H       = normalize(V + L);
    vec3 radiance = lightColor[i] * attenuation(i);

    // Cook-Torrance BRDF
    float D = D_GGX(N, H, alpha);
    vec3  F = F_Schlick(max(dot(V, H), 0.0), F0);
    float G = G_Smith(N, V, L, roughness);

    vec3 kS      = F;
    vec3 kD      = (vec3(1.0) - kS) * (1.0 - metallic);

    vec3 diffuse  = kD * baseColor / PI;
    vec3 specular = (D * F * G) / max(4.0 * dot(N, L) * dot(N, V), 0.001);

    Lo += (diffuse + specular) * radiance * max(dot(N, L), 0.0);
}

// -------- IBL Indirect Lighting --------
vec3 F_ibl = F_SchlickRoughness(max(dot(N, V), 0.0), F0, roughness);
vec3 kS_ibl = F_ibl;
vec3 kD_ibl = (1.0 - kS_ibl) * (1.0 - metallic);

// Diffuse IBL: sample irradiance map
vec3 irradiance = texture(irradianceMap, N).rgb;
vec3 indDiffuse = kD_ibl * baseColor * irradiance;

// Specular IBL: prefiltered env map + BRDF LUT (Split-Sum)
const float MAX_REFLECTION_LOD = 4.0;
vec3 prefilteredColor = textureLod(prefilterMap, R, roughness * MAX_REFLECTION_LOD).rgb;
vec2 brdfLUT = texture(brdfLUTTex, vec2(max(dot(N, V), 0.0), roughness)).rg;
vec3 indSpecular = prefilteredColor * (F0 * brdfLUT.x + brdfLUT.y);

vec3 ambient = indDiffuse + indSpecular;
vec3 color   = ambient + Lo;
```

## Disney BRDF 原则

迪士尼在推出 PBR 流水线时，提出了 **Disney BRDF Principle**，不仅是技术规范，更是材质系统的设计哲学：

1. **参数要符合艺术家的直觉**
   - 不能暴露"光子吸收截面"、"电子跃迁频率"等物理概念
   - 参数名要艺术家能理解：roughness（粗糙度）、metallic（金属度）

2. **参数要尽可能少**
   - 参数太多导致相互影响，艺术家调参时"按下葫芦浮起瓢"

3. **所有参数值的范围最好在 0~1 之间**
   - Blinn-Phong 的 shininess 参数可以是任意正数，艺术家不知道 100 和 50 的关系
   - PBR 参数归一化后，0 = 完全无，1 = 完全有，直觉清晰

4. **允许超出 [0,1] 范围，但必须产生有意义的效果（创作弹性）**

5. **所有参数组合必须物理合理**

这些原则后来成为整个游戏行业 PBR 材质系统的设计准则。

## 光照计算

### 直接光照（Direct Lighting）

直接光照计算相对清晰：对场景中有限个光源，各自计算 Cook-Torrance BRDF，加上 Shadow Map 可见性判断：

$$L_{direct} = \sum_{k} \text{visibility}(k) \cdot f_r(\omega_k, \omega_o) \cdot L_k \cdot (\mathbf{n} \cdot \omega_k)$$

### 间接光照（Indirect Lighting / Global Illumination）

间接光照是渲染效果的关键，差异巨大：
- 无间接光照：室内场景平面感强烈，光只照到直接受光处
- 有间接光照：光照到地面弹到墙壁，整个空间饱满自然

全局光照（GI）的挑战：需要对球面光照函数做卷积积分，实时计算量极大。

解决方案演化历程：

1. **Ambient 常数** → 最简单，但平面感强
2. **Spherical Harmonics（SH）预计算** → 低频 GI
3. **Light Map 烘培** → 高质量静态 GI
4. **Light Probe（反射探头）** → 动态物体的 GI 近似
5. **IBL（Image-Based Lighting）** → 完整的环境光照

## 球谐函数（Spherical Harmonics）

**Spherical Harmonics（SH）** 是求解球面函数的一组正交基函数，由 19 世纪数学家傅里叶的思想延伸而来（球面上的"傅里叶变换"）。

由 Peter-Pike Sloan（微软 DirectX 团队）将 SH 引入实时渲染，提出 **PRT（Precomputed Radiance Transfer）**。

### SH 的关键性质

- **完备正交基**：无数个 SH 基函数可表示任意球面函数，类似傅里叶展开
- **光滑性**：每个 SH 基函数的二阶导恒为零，函数永远光滑连续
- **正交性**：任意两个不同阶 SH 函数的卷积（内积）为零

### 低阶 SH 近似

游戏中通常只用 **L0 + L1（共 4 个系数）或 L2（共 9 个系数）**：

- **L0（0 阶）**：1 个基函数，球面均值，代表整体亮度
- **L1（1 阶）**：3 个基函数，对应笛卡尔坐标 X、Y、Z，表示光的主要方向

4 个参数 = 4 次点乘就能查询任意方向的环境光照。

**SH 的局限**：只能表示低频光照（软阴影、天空光、环境散射），高频细节（直接高光、锐利反射）需要其他手段。但对于间接漫反射，SH 已经足够。

## IBL 图像基础光照

**IBL（Image-Based Lighting）** 利用预滤波的环境贴图（Cube Map）来表示来自四面八方的光照，是现代游戏渲染的核心技术。

### 基本思路

渲染方程中的间接光照积分：

$$L_{indirect}(x, \omega_o) = \int_{\Omega^+} f_r(\omega_i, \omega_o) \cdot L_{env}(\omega_i) \cdot (\mathbf{n} \cdot \omega_i)\, d\omega_i$$

直接计算需要对每个像素采样球面上 N 个点，成本极高（100 个采样点 × 数百万像素 × 30 帧/秒 = 无法承受）。

IBL 的核心思路：**预计算（空间换时间）**——离线预处理环境贴图，运行时查表。

### Diffuse IBL：辐照度贴图（Irradiance Map）

漫反射部分 BRDF = $c/\pi$，代入积分：

$$L_{diffuse}(x) = \frac{c}{\pi} \int_{\Omega^+} L_{env}(\omega_i)\, (\mathbf{n} \cdot \omega_i)\, d\omega_i$$

积分结果只与法向 $\mathbf{n}$ 有关，与视角无关！

预处理方法：将原始 Cube Map 与 Lambertian 核函数（$\cos\theta$）进行卷积，生成 **Irradiance Map**（辐照度贴图）。

运行时：以表面法向 $\mathbf{n}$ 采样 Irradiance Map，直接得到漫反射光照。

### Specular IBL：预滤波环境贴图（Prefiltered Environment Map）

镜面反射积分更复杂，因为它与视角方向 $\mathbf{v}$ 相关（通过半程向量 $\mathbf{h}$）：

$$L_{specular}(x, \mathbf{v}) = \int_{\Omega^+} f_{spec}(\omega_i, \mathbf{v})\, L_{env}(\omega_i)\, (\mathbf{n} \cdot \omega_i)\, d\omega_i$$

**Split-Sum 近似**（Epic Games 提出）——将积分拆分为两个独立的查表操作：

$$L_{specular} \approx \underbrace{\left(\int_{\Omega^+} L_{env}(\omega_i)\, d\omega_i\right)}_{\text{预滤波环境贴图}} \cdot \underbrace{\left(\int_{\Omega^+} f_{spec}(\omega_i, \mathbf{v})\, (\mathbf{n} \cdot \omega_i)\, d\omega_i\right)}_{\text{BRDF LUT}}$$

#### 预滤波环境贴图（Prefiltered Environment Map）

按 roughness 预先对 Cube Map 做不同程度的模糊，存入不同的 **Mip 层级**：

- Mip 0：roughness=0，原始清晰图（完美镜面）
- Mip 1：roughness=0.25，轻微模糊
- Mip 2：roughness=0.5，中等模糊
- Mip 3：roughness=0.75，较模糊
- Mip 4：roughness=1.0，完全模糊（漫反射极限）

> **重要**：这些 Mip 层每一层需要用 GGX 核单独卷积计算，**不是**硬件自动生成的标准 Mip（降采样）！

运行时：以反射方向 $\mathbf{R} = \text{reflect}(-\mathbf{v}, \mathbf{n})$ 和 roughness 作为索引，用 `textureLod` 采样。

#### BRDF LUT（查找表）

第二个积分 $\int f_{spec}(\omega_i, \mathbf{v})\, (\mathbf{n} \cdot \omega_i)\, d\omega_i$ 预计算成一张 2D 查找表：

- **X 轴**：$\cos\theta = \mathbf{n} \cdot \mathbf{v}$（法线与视角夹角）
- **Y 轴**：roughness
- **输出**：R 通道 = 缩放系数，G 通道 = 偏置系数

运行时：

```glsl
vec2 brdf = texture(brdfLUT, vec2(NdotV, roughness)).rg;
vec3 specular = prefilteredColor * (F0 * brdf.x + brdf.y);
```

这样，整个 IBL 镜面反射只需要两次纹理采样。

## 全局光照解决方案

### Light Map（光照贴图）烘培

**Light Map** 是将场景静态光照（包括全局光照效果）预先烘培到 2D 纹理的技术。

#### 工作流程

1. **UV 展开（UV Parameterization / Unwrapping）**：将 3D 网格展开到 2D，生成 Atlas
   - 要求每个 texel 对应的世界空间面积尽量均匀（防止部分区域过于密集/稀疏）
2. **Light Farm 烘培**：使用离线渲染集群（多台高性能服务器），在 Light Map 的每个 texel 上做路径追踪
3. **存储**：通常用 BC6H 等压缩格式存储，每个地图几十到几百 MB

#### 优点

- **运行时成本极低**：只需采样一张纹理
- **质量极高**：可以模拟多次弹射的间接光照、软阴影、Color Bleeding
- 艺术家控制度高，效果稳定

#### 缺点

- **烘培时间极长**：大场景需要半小时到数小时，影响迭代效率
- **仅支持静态物体和静态光源**：动态物体和动态光无法使用
- **存储占用大**：大场景可达数百 MB
- 动态物体与 Light Map 的融合存在"漏光"问题（人物走到 Light Map 投影的阴影区域时突然变暗）

### Light Probe（光照探针）

**Light Probe** 是在场景中放置"采样点"，捕获该位置的全向光照信息（通常以 SH 或 Cube Map 存储），供动态物体使用。

#### 工作原理

- 在场景中布置一个稀疏的探针网格
- 每个探针离线烘培或运行时更新其位置的 SH 系数
- 运行时，动态物体根据位置在周围探针间插值（四面体插值/Delaunay 插值）

#### 优缺点

- **优点**：支持动态物体；运行时成本低
- **缺点**：采样密度远低于 Light Map（成千上万 vs 数百万），细节差；光照精度受探针密度限制

#### Light Map + Light Probe 组合方案

在过去十几年的 3A 游戏中，这是标准方案：
- **静态场景**：用 Light Map 获得高质量 GI
- **动态物体**：用 Light Probe 近似 GI

## 材质系统实现

### PBR 材质工作流总结

```
输入 Textures:
  ├── BaseColor Map   (albedo/漫反射颜色)
  ├── Metallic Map    (金属度)
  ├── Roughness Map   (粗糙度)
  ├── Normal Map      (切线空间法线)
  ├── AO Map          (环境遮蔽)
  └── Emissive Map    (自发光，可选)

运行时计算:
  ├── F0 = mix(0.04, baseColor, metallic)
  ├── alpha = roughness^2
  │
  ├── 直接光照:
  │   ├── D = D_GGX(N, H, alpha)
  │   ├── F = F_Schlick(dot(V,H), F0)
  │   ├── G = G_Smith(N, V, L, roughness)
  │   ├── specular = D*F*G / (4*NdotL*NdotV)
  │   └── diffuse = (1-F)*(1-metallic)*baseColor/PI
  │
  └── 间接光照 (IBL):
      ├── diffuse = kD * baseColor * sample(irradianceMap, N)
      └── specular = sample(prefilterMap, R, roughness*LOD)
                   * (F0 * brdfLUT.x + brdfLUT.y)
```

### 混合光照的实用 Hack

工业级游戏引擎（包括讲师自己的引擎）的实际实现通常是多种技术的混合：

1. **环境光**：SH 探针（低频 GI，运行时查询）
2. **主光源**：方向光 + Shadow Map（高质量直接光照）
3. **高频镜面**：Prefiltered Environment Map + BRDF LUT（IBL specular）
4. **静态 GI**：Light Map（高质量间接光照，仅静态物体）
5. **动态物体**：Light Probe 插值（近似 GI）

将这些叠加后，已经可以满足卡基亚渲染方程的大部分积分项：
- 直接光近似：主光源
- 低频间接光：SH Ambient
- 高频反射：Environment Map
- 完整 GI：Light Map

## 关键术语表

| 术语 | 英文 | 含义 |
|------|------|------|
| **辐亮度** | Radiance ($L$) | 沿特定方向单位立体角单位投影面积的辐射功率 |
| **辐照度** | Irradiance ($E$) | 单位面积接收的总辐射功率，= $\int L \cos\theta \, d\omega$ |
| **BRDF** | Bidirectional Reflectance Distribution Function | 双向反射分布函数，描述材质散射特性 |
| **渲染方程** | Rendering Equation | Kajiya 1986，完整描述表面光照的方程 |
| **Lambert 余弦律** | Lambert's Cosine Law | 表面接收能量与入射角余弦成正比 |
| **能量保守** | Energy Conservation | 出射能量不超过入射能量（BRDF 约束） |
| **PBR** | Physically Based Rendering | 基于物理的渲染，使用微平面理论等物理模型 |
| **微平面理论** | Microfacet Theory | 表面由微小镜面组成，其法向分布决定材质外观 |
| **NDF** | Normal Distribution Function | 法线分布函数（Cook-Torrance 的 D 项） |
| **GGX** | — | Trowbridge-Reitz NDF，工业界标准，特点：尖峰+长尾 |
| **菲涅尔效应** | Fresnel Effect | 掠射角时反射率急剧增加的光学现象 |
| **Schlick 近似** | Schlick Approximation | 菲涅尔的五次方近似公式 |
| **F0** | Fresnel at 0° | 法线方向入射时的菲涅尔反射率 |
| **Smith 几何项** | Smith Geometry | 微平面遮蔽/阴影的近似公式（G 项） |
| **IBL** | Image-Based Lighting | 图像基础光照，用环境贴图表示来自四面八方的光 |
| **Irradiance Map** | 辐照度贴图 | 对 Cube Map 用 Lambertian 核卷积后的结果，用于漫反射 IBL |

| **BRDF LUT** | BRDF 查找表 | Split-Sum 中 BRDF 积分的预计算查找表 |
| **Split-Sum** | — | Epic Games 将镜面 IBL 积分拆成两个独立查表的近似方法 |
| **Light Map** | 光照贴图 | 预烘培的全局光照纹理，存储在场景 UV 展开后的 Atlas 上 |
| **Atlas** | 图集 | 将多个几何体的 UV 展开拼合在一张纹理上 |
| **Light Probe** | 光照探针 | 场景中采样点，捕获该位置全向光照，供动态物体使用 |
| **PRT** | Precomputed Radiance Transfer | 预计算辐亮度传输，Peter-Pike Sloan 提出，SH 加速技术 |
| **SH** | Spherical Harmonics | 球谐函数，球面上的正交基函数集，用于近似低频光照 |
| **GI** | Global Illumination | 全局光照，包含间接光照的完整光照系统 |
| **Shadow Map** | 阴影贴图 | 从光源角度渲染深度图来判断阴影的经典算法 |

## 总结

本节课以 Kajiya 渲染方程为主线，讲解了游戏实时渲染从"经典三件套"到现代 PBR 的发展历程：

**核心知识脉络：**

1. **渲染方程**：$L_o = L_e + \int f_r \cdot L_i \cdot \cos\theta \, d\omega_i$，是所有渲染技术的理论基础

2. **经典方案**：Ambient + 主光源 + Environment Map + Blinn-Phong，简单有效，至今低端机仍可用

3. **PBR 革命**：微平面理论 + Cook-Torrance BRDF（D/F/G 三项）+ Disney 原则，奠定现代材质系统

4. **GI 方案**：SH 低频 GI + Light Map 静态 GI + Light Probe 动态 GI，三者组合覆盖不同需求

5. **IBL**：预滤波 Cube Map + BRDF LUT + Split-Sum，用空间换时间解决球面积分实时化难题

**讲师的核心感悟：**
> 过去几十年，全世界最聪明的人把各种大胆的 hack 拼在一起，没有严格推导，全靠经验和直觉——最后居然都 work 了，造就了我们今天享受的游戏视觉体验。

这正是游戏图形学的魅力所在。
