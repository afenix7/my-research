## 1. 概述

本集讲解如何修改 UE5.1 引擎源码，在现有延迟渲染管线的末尾插入一个自定义的 **Screen Pass**（屏幕后处理 Pass）。

核心思路：
- 整个渲染现象（Deferred Rendering）由若干 Pass 串联完成。
- **Screen Pass** 是其中一种特殊 Pass，它直接针对最终合成好的屏幕画面执行像素着色器（Pixel Shader），本质上是一段 **Post Process Effect**。
- 典型应用：描边（Laplacian）、高斯模糊、色调映射、泛光等。

与业务层 Material 的 Post Process Volume 不同，源码级 Screen Pass 直接嵌入渲染管线，优先级和灵活性更高，且不依赖任何 UObject 框架。

## 2. 屏幕特效原理

### 2.1 Post Process Effect 在渲染管线中的位置

```
G-Buffer 生成
   ↓
光照计算（Lighting Pass）
   ↓
透明物体（Translucency）
   ↓
Tone Mapping / 后处理链
   ↓
[ 自定义 Screen Pass（本集内容）]
   ↓
输出到 Back Buffer
```

### 2.2 Screen Pass 本质

Screen Pass 的本质是对一张已经渲染完毕的 Render Target 执行一次 **全屏 Pixel Shader**：

- **输入**：已渲染好的场景颜色贴图（Scene Color Texture）+ 采样器（Sampler）
- **输出**：同一块 Render Target（或新建的 RT）
- **着色器**：一段标准 HLSL Pixel Shader，逐像素处理

### 2.3 RDG（Render Dependency Graph）

UE5 使用 RDG 管理所有渲染资源和 Pass 的依赖关系。添加 Screen Pass 也需遵循 RDG 的注册方式：

- 使用 `FRDGBuilder` 创建贴图（`CreateTexture`）
- 使用 `AddCopyPass` 拷贝现有 Render Target
- 使用 `AddScreenPass` 注册自定义屏幕 Pass

## 3. 实战目标：拉普拉斯描边与模糊

### 3.1 拉普拉斯算子（Laplacian）—— 描边效果

拉普拉斯算子对图像中每个像素取其与周围 8 个邻居像素的加权差：

| 位置 | 权重 |
|------|------|
| 中心像素 | +8 |
| 周围 8 个像素 | -1（每个）|

计算结果若接近 0（平坦区域）则为黑色；在边缘处差值大则为高亮。这样就产生了**黑白描边**的效果。

加权和公式：

```
result = 8 * center - (top + bottom + left + right + topLeft + topRight + bottomLeft + bottomRight)
```

### 3.2 均匀模糊（Box Blur）—— 模糊效果

将所有 9 个像素（中心 + 周围 8 个）权重均设为 `1/9`，即简单均值滤波，产生**模糊效果**：

```
result = (center + top + bottom + left + right + topLeft + topRight + bottomLeft + bottomRight) / 9.0
```

## 4. 完整实现步骤

### Step 1：编写 USF Pixel Shader

**文件位置：**

```
Engine/Shaders/Private/MyScreenPass.usf
```

> 注意：USF 文件必须放在 `Engine/Shaders/Private/` 目录下，这是 UE5 引擎 Shader 文件的规范位置。

参照引擎现有的 USF 文件格式编写，声明输入纹理和采样器，实现 Pixel Shader 主函数（通常命名为 `MainPS`）。

### Step 2：在 C++ 中继承 FGlobalShader

在引擎渲染模块（如 `Renderer` 模块）的某个 `.cpp` 文件中：

1. 声明一个继承自 `FGlobalShader` 的类
2. 用宏声明着色器参数（纹理 + 采样器）
3. 用 `IMPLEMENT_GLOBAL_SHADER` 宏注册，指定 USF 路径和入口函数名

### Step 3：封装 RenderScreenPass 函数

在对应头文件中声明函数，在 cpp 中实现：

1. 创建一张新的 RDG Texture（用于接收拷贝结果）
2. 调用 `AddCopyPass` 将当前 Scene Color RT 拷贝到新贴图
3. 填充着色器参数（输入纹理 = 拷贝出来的新贴图，采样器）
4. 调用 `AddScreenPass` 执行自定义 Pixel Shader

> 关键原因：当前 Render Target 不能同时作为输入和输出，因此必须先拷贝一份，再以拷贝作为输入。

### Step 4：在 DeferredShadingRenderer 中调用

修改文件：

```
Engine/Source/Runtime/Renderer/Private/DeferredShadingRenderer.cpp
```

在该文件的 `Render()` 函数末尾（约第 3900 行）遍历所有 View，逐 View 调用上一步封装的函数。

## 5. 关键代码

### HLSL Shader（USF）

文件：`Engine/Shaders/Private/MyScreenPass.usf`

```hlsl
#include "/Engine/Public/Platform.ush"

// 输入参数：由 C++ 侧传入
Texture2D InputTexture;
SamplerState InputSampler;

// Pixel Shader 主函数
float4 MainPS(float4 SvPosition : SV_Position) : SV_Target
{
    // 当前像素的 UV（需要结合 View.BufferSizeAndInvSize 计算，这里简化演示）
    float2 UV = SvPosition.xy * View.BufferSizeAndInvSize.zw;

    // 像素偏移量（采样周围 8 个邻居）
    float2 TexelSize = View.BufferSizeAndInvSize.zw;

    // 采样中心及周围 8 个像素
    float4 center      = Texture2DSample(InputTexture, InputSampler, UV);
    float4 top         = Texture2DSample(InputTexture, InputSampler, UV + float2( 0,  1) * TexelSize);
    float4 bottom      = Texture2DSample(InputTexture, InputSampler, UV + float2( 0, -1) * TexelSize);
    float4 left        = Texture2DSample(InputTexture, InputSampler, UV + float2(-1,  0) * TexelSize);
    float4 right       = Texture2DSample(InputTexture, InputSampler, UV + float2( 1,  0) * TexelSize);
    float4 topLeft     = Texture2DSample(InputTexture, InputSampler, UV + float2(-1,  1) * TexelSize);
    float4 topRight    = Texture2DSample(InputTexture, InputSampler, UV + float2( 1,  1) * TexelSize);
    float4 bottomLeft  = Texture2DSample(InputTexture, InputSampler, UV + float2(-1, -1) * TexelSize);
    float4 bottomRight = Texture2DSample(InputTexture, InputSampler, UV + float2( 1, -1) * TexelSize);

    // ----- 拉普拉斯算子（描边效果）-----
    float4 laplacian = 8.0 * center
        - top - bottom - left - right
        - topLeft - topRight - bottomLeft - bottomRight;

    // 转灰度（去掉彩色，仅保留边缘强度）
    float gray = dot(laplacian.rgb, float3(0.299, 0.587, 0.114));
    float4 outColor = float4(gray, gray, gray, 1.0);

    // ----- 若要改成均匀模糊，替换为以下代码 -----
    // float4 outColor = (center + top + bottom + left + right
    //                  + topLeft + topRight + bottomLeft + bottomRight) / 9.0;

    return outColor;
}
```

> 说明：
> - `View.BufferSizeAndInvSize` 是 UE5 内置的 ViewUniformBuffer 参数，`.zw` 分量为逆分辨率（即像素偏移 1 个像素的 UV 步长）。
> - 转灰度行是描边效果特有的一步，若要保留彩色描边可直接用 `laplacian`。
> - 切换为模糊效果只需注释掉 laplacian 那段，启用 box blur 那段。

### C++ FGlobalShader 子类

```cpp
// MyScreenPassShader.h（或直接写在 .cpp 中）

#include "GlobalShader.h"
#include "ShaderParameterStruct.h"
#include "ScreenPass.h"

// Pixel Shader 类
class FMyScreenPassPS : public FGlobalShader
{
public:
    DECLARE_GLOBAL_SHADER(FMyScreenPassPS);
    SHADER_USE_PARAMETER_STRUCT(FMyScreenPassPS, FGlobalShader);

    // 声明着色器参数
    BEGIN_SHADER_PARAMETER_STRUCT(FParameters, )
        SHADER_PARAMETER_RDG_TEXTURE(Texture2D, InputTexture)          // 输入纹理
        SHADER_PARAMETER_SAMPLER(SamplerState, InputSampler)           // 采样器
        SHADER_PARAMETER_STRUCT_REF(FViewUniformShaderParameters, View) // View 参数（含 BufferSize 等）
        RENDER_TARGET_BINDING_SLOTS()                                   // 输出 RT 绑定槽
    END_SHADER_PARAMETER_STRUCT()
};

// 注册 Shader
// 注意路径格式："/Engine/Private/MyScreenPass.usf"
// 对应磁盘路径：Engine/Shaders/Private/MyScreenPass.usf
// 规则：注册路径去掉 "Shaders" 层级
IMPLEMENT_GLOBAL_SHADER(FMyScreenPassPS,
    "/Engine/Private/MyScreenPass.usf",  // USF 虚拟路径（不含 Shaders 目录层）
    "MainPS",                             // Pixel Shader 入口函数名
    SF_Pixel);
```

> 路径规则重点：
> - 磁盘路径：`Engine/Shaders/Private/MyScreenPass.usf`
> - 注册路径：`/Engine/Private/MyScreenPass.usf`（去掉了 `Shaders` 目录层）
> - 这是 UE5 内置规则，不可更改。

### RDG Screen Pass 调用函数

在 `DeferredShadingRenderer.h` 中声明：

```cpp
void RenderMyScreenPass(FRDGBuilder& GraphBuilder, const FViewInfo& View, FRDGTextureRef SceneColorTexture);
```

在 `DeferredShadingRenderer.cpp`（或新增的 `.cpp` 文件）中实现：

```cpp
void FDeferredShadingSceneRenderer::RenderMyScreenPass(
    FRDGBuilder& GraphBuilder,
    const FViewInfo& View,
    FRDGTextureRef SceneColorTexture)
{
    // 1. 创建一张临时贴图，用于拷贝当前 Scene Color
    //    原因：Render Target 不能同时作为输入和输出
    FRDGTextureDesc CopyDesc = SceneColorTexture->Desc;
    FRDGTextureRef CopiedTexture = GraphBuilder.CreateTexture(CopyDesc, TEXT("MyScreenPassInputCopy"));

    // 2. 添加 Copy Pass：把当前 Scene Color 拷贝到 CopiedTexture
    AddCopyTexturePass(GraphBuilder, SceneColorTexture, CopiedTexture);

    // 3. 构造 Pixel Shader 参数
    FMyScreenPassPS::FParameters* PassParameters = GraphBuilder.AllocParameters<FMyScreenPassPS::FParameters>();
    PassParameters->InputTexture = CopiedTexture;                          // 输入为拷贝出来的贴图
    PassParameters->InputSampler = TStaticSamplerState<SF_Bilinear>::GetRHI();
    PassParameters->View         = View.ViewUniformBuffer;
    PassParameters->RenderTargets[0] = FRenderTargetBinding(SceneColorTexture, ERenderTargetLoadAction::ELoad);

    // 4. 获取 Shader
    TShaderMapRef<FMyScreenPassPS> PixelShader(View.ShaderMap);

    // 5. 添加 Screen Pass（全屏四边形 Draw Call）
    FScreenPassViewport Viewport(View.ViewRect);
    AddDrawScreenPass(
        GraphBuilder,
        RDG_EVENT_NAME("MyCustomScreenPass"),
        View,
        Viewport,
        Viewport,
        PixelShader,
        PassParameters
    );
}
```

### DeferredShadingRenderer 调用点

文件：`Engine/Source/Runtime/Renderer/Private/DeferredShadingRenderer.cpp`

在主渲染函数 `FDeferredShadingSceneRenderer::Render()` 末尾（约第 3900 行）添加：

```cpp
// ---- 在延迟渲染最终输出之后，调用自定义 Screen Pass ----
// 对所有 View 分别执行（多视口、分屏等场景）
for (int32 ViewIndex = 0; ViewIndex < Views.Num(); ViewIndex++)
{
    const FViewInfo& View = Views[ViewIndex];
    RenderMyScreenPass(GraphBuilder, View, SceneTextures.Color.Target);
}
```

> 位置选择原则：
> - 必须在延迟渲染已将最终画面写入 Scene Color 之后调用。
> - 该函数（`Render()`）约从第 2000 行到第 4000 行，共 2000 行；插入点在约 3900 行处。
> - 后续分析整个 `Render()` 函数可了解完整渲染流程。

## 7. 调试方法

### 7.1 实时热重载 Shader

修改 USF 后，不必重新编译整个引擎，在编辑器中按 `Ctrl+Shift+.`（或在控制台输入 `recompileshaders`）可热重载着色器。

修改 C++ 代码（FGlobalShader 子类、Pass 调用逻辑）则需要重新编译 Renderer 模块。

### 7.2 RenderDoc 抓帧

使用 RenderDoc 或 UE5 内置的 GPU Insight 可以：
- 查看自定义 Pass 是否被插入渲染序列
- 检查输入贴图内容是否正确
- 查看 Pixel Shader 逐像素输出

在 `AddDrawScreenPass` 中通过 `RDG_EVENT_NAME("MyCustomScreenPass")` 给 Pass 命名，在 RenderDoc 的帧捕获里可以按名字定位。

### 7.3 视觉验证

教程中演示了直接启动编辑器（Debug 模式），效果立竿见影：
- 拉普拉斯算子：整个场景变为黑白描边风格
- 均匀模糊：整个场景产生轻微模糊效果

注意：教程中提到屏幕上会有一块白色区域，那是讲师额外写的 Debug 输出，不属于本功能。

### 7.4 参数 Debug 输出

可以在 Pixel Shader 里临时输出特定通道：

```hlsl
// 输出纯红色，验证 Pass 是否被调用
return float4(1, 0, 0, 1);
```

## 9. 扩展：更换算子实现不同效果

只需修改 USF 文件中的权重逻辑，无需改动任何 C++ 代码：

| 算子 | 效果 | 中心权重 | 邻居权重 |
|------|------|---------|---------|
| 拉普拉斯 | 黑白描边 | +8 | 各 -1 |
| Box Blur（均匀模糊） | 均匀模糊 | 1/9 | 各 1/9 |
| 高斯模糊 | 自然模糊 | 中心权重更大 | 按高斯分布递减 |
| Sobel | 带方向的边缘检测 | 0 | 两组 3x3 核 |

高斯模糊的权重与均匀模糊不同——中间像素权重更大，周围像素权重按距离衰减（符合正态分布），产生更自然的模糊效果。

## 10. 小结

本集演示了在 UE5.1 引擎源码中插入自定义 Screen Pass 的完整流程，要点如下：

1. **Screen Pass 的本质**是全屏 Pixel Shader，针对已渲染完毕的画面做后处理。
2. **三步走**：写 USF Shader → 继承 `FGlobalShader` → 在 DeferredShadingRenderer 中插入调用。
3. **拷贝贴图是必须的**：同一 Render Target 不能同时充当输入和输出，需通过 `AddCopyPass` 先拷贝。
4. **路径规则**：USF 注册路径省略 `Shaders/` 层级（这是 UE5 内置约定）。
5. **插入点**：`DeferredShadingRenderer::Render()` 约第 3900 行，位于所有延迟渲染步骤之后。
6. **`DeferredShadingRenderer.cpp` 的 `Render()` 函数**（约 2000~4000 行）是理解 UE5 渲染流程的关键入口，后续系列将深入分析。

通过更换 Shader 中的卷积核权重，可以轻松实现描边、模糊、锐化等各类屏幕特效，而无需修改任何 C++ 渲染逻辑。
