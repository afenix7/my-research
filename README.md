# Research: Agent Implementation Comparison

Comparative research on how **four leading AI coding agent projects** implement:

- **Agent React pattern** - the core reaction loop
- **Context management & context compression** - keeping context within token limits
- **SubAgent creation** - spawning child agents for subtasks
- **Skill mechanism** - reusable sharable skills/instructions
- **Agent session isolation** - forking/branching/experimentation
- **(Claude Code only)** Agent Teams - multi-agent parallel collaboration

## Comparison Table Summary

| Feature | pi-mono | OpenCode | Codex CLI | Claude Code |
|---------|---------|----------|-----------|------------|
| **Agent React** | Event-driven double loop | ACP + event bus | Simple ReAct | ReAct + hierarchical Team Lead/Member |
| **Context Compression** | LLM structured summary + keep recent | Pruning + LLM compression (2-level) | Sliding window + summary | Auto-compaction with circuit breaker |
| **SubAgent** | Markdown discovery | Config + dynamic generation, permission isolation | Process-level spawn | Built-in `spawn_sub_agent` + Agent Teams |
| **Skill Format** | `SKILL.md` Markdown | `SKILL.md` + remote Git | MCP server config | `SKILL.md` + native MCP |
| **Session Isolation** | Tree structured file (line-delimited JSON) | SQL + parent_id reference | File-level isolation | git worktree + branching |
| **Agent Teams** | ❌ | ❌ | ❌ | ✅ Native parallel collaboration |
